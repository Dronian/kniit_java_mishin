### Лабораторная работа # 4: Менеджер паролей

Создайте RESTful веб-приложение "Менеджер паролей" на Spring Boot. 

Приложение позволяет пользователям управлять своими паролями: добавлять, просматривать, изменять и удалять их. 
Пароли хранятся в базе данных PostgreSQL в зашифрованном виде с возможностью расшифровки для показа пользователю.

#### **Требования к функционалу:**
Эндпоинты API (Spring MVC):
- `POST /passwords` — добавить пароль.
- `GET /passwords` — список паролей пользователя - возвращает только id, site, username. Можно показать пароль в скрытом виде `*********`
- `GET /passwords/{id}` — один пароль с расшифровкой.
- `PUT /passwords/{id}` — обновить пароль
- `DELETE /passwords/{id}` — удалить пароль.

#### **Ограничения:**
- Пользователь может получить доступ только к своим паролям, для простоты передаем идентификатор пользователя (X-User-Id: 123) в заголовке запроса.
- Если пользователь попытается получить доступ к чужим паролям, должен вернуться ответ со статусом 403 Forbidden и сообщением об ошибке.
- Если пароль не найден, вернуть ответ со статусом 404 Not Found и сообщением об ошибке.
- Шифрование: Сохранять пароли зашифрованными, расшифровывать только при показе полного пароля. Для реализации данного функционала нужно применить алгоритм AES - он позволяет расшифровывать.
- Архитектура: в коде должны быть явно выделены слои контроллеров, сервисов и DAO.
- Пользователи и их пароли должны хранится в разных таблицах БД. Должна быть реализована связь один ко многим.

#### **Технологический стек**
- Spring Boot (Web, Data JPA, Security)
- Spring MVC (REST)
- Spring Data JPA + PostgreSQL (можно заменить другой реляционной БД)
- Java `javax.crypto` для AES-шифрования
- Сборка с Maven
- Модульные тесты JUnit, Mockito

#### **Схема работы приложения**
Шифрование при сохранении:
```
Пользователь отправляет plaintext-пароль → шифруете AES → храните в БД.
```
Расшифровка при показе:
```
Извлекаете зашифрованный пароль из БД → расшифровываете AES с ключом пользователя → возвращаете plaintext (или маску для GET-списка).
```

#### **Примечание**
Разрабатывайте приложение поступательно: сначала реализуйте функционал добавления/удаления/изменения/просмотра пользователей и паролей, и только потом добавляйте шифрование.
